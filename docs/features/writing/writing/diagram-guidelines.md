# Diagram Guidelines

**Epic 7 - Story 7.7**

## Purpose
Ensure diagrams reduce cognitive load rather than perform. Use sparingly, only when words alone are insufficient.

---

## Core Principle

**A diagram should make something easier to understand, not make you look smart.**

Use diagrams only when:
- The concept is inherently spatial or relational
- A visual representation reduces cognitive load
- Words alone would require significant mental model building

Don't use diagrams to:
- Make the essay look more professional
- Break up text
- Show off design skills
- Make simple concepts seem complex

---

## Maximum: One Diagram Per Essay

**Rule:** Max 1 diagram per essay.

**Why:**
- Forces you to choose what's truly essential
- Prevents visual clutter
- Maintains library feel (not slide deck)
- Ensures text works standalone

**If you think you need more than one diagram:**
1. Can these be combined into a single diagram?
2. Is one diagram more essential than the other?
3. Can one be explained in text instead?
4. Should this be split into multiple essays?

## Style Rules

### Hand-Drawn Aesthetic

**Preferred:**
- Excalidraw or similar hand-drawn tools
- Simple boxes, circles, arrows
- Clean but informal
- Black, white, and one accent color (indigo)

**Why:**
- Matches professional record tone (not sales deck)
- Focuses on content, not polish
- Feels like whiteboard explanation
- Faster to create, easier to update

### Visual Constraints

**Use:**
- Thin lines (1-2px)
- Minimal color (black + indigo accent)
- Simple shapes (boxes, circles, arrows)
- Clear labels (sans-serif, readable size)
- White or off-white background

**Avoid:**
- Gradients or shadows
- Multiple colors for decoration
- Complex shapes or icons
- Decorative fonts
- Textures or patterns
- 3D effects

### Label Discipline

**Labels should be:**
- Short (1-5 words)
- Specific (not generic)
- Readable (sufficient size)
- Minimal (only what's necessary)

**Good labels:**
- "Student submission"
- "AI analysis engine"
- "Framework matching"
- "Personalized feedback"

**Bad labels:**
- "Input" (too generic)
- "Super advanced machine learning-powered artificial intelligence system for automated essay evaluation and feedback generation" (too long)
- "ğŸ¯ Magic happens here!" (decorative, not informative)

## Diagram Types

### âœ… Allowed Types

**1. System Diagrams**
- Components and connections
- Data flow
- Architecture overview

**When to use:** Explaining how a system works

**Example context:** "MarkPoint's feedback engine has three stages..."

**2. Process Flows**
- Sequential steps
- Decision points
- Branching paths

**When to use:** Explaining a workflow or decision process

**Example context:** "How I evaluate whether to build a feature..."

**3. Relationship Maps**
- Hierarchies
- Dependencies
- Conceptual connections

**When to use:** Showing how concepts relate

**Example context:** "The relationship between teaching methods, student outcomes, and retention..."

**4. Comparison Diagrams**
- Before/after
- Old vs. new
- Approach A vs. Approach B

**When to use:** Illustrating change or contrast

**Example context:** "Old: Content-first teaching. New: Framework-first teaching."

**5. Conceptual Models**
- Abstract frameworks
- Mental models
- Tradeoff triangles

**When to use:** Explaining how you think about a problem

**Example context:** "The speed-depth-breadth triangle: you can optimize for two..."

### âŒ Forbidden Types

**1. Stock Photos or Generic Icons**
- No people looking at laptops
- No lightbulb "ideas"
- No puzzle pieces "coming together"
- No handshake "partnerships"

**Why:** Performative, not explanatory

**2. Data Visualizations**
- No bar charts of vanity metrics
- No growth curves projecting upward
- No pie charts without specific data

**Why:** Should be in text with context, not standalone visuals

**Exception:** If you have real data that's essential to understanding, present it as simple table in text

**3. Mockups or Screenshots**
- No polished product mockups
- No marketing screenshots
- No staged interface examples

**Why:** This is writing, not a portfolio. If you need to show a UI, reference your Work section instead.

**4. Infographics**
- No "10 steps to..." posters
- No text-heavy visual layouts
- No decorative data presentations

**Why:** If it needs that much text, write it as text

**5. Animations or GIFs**
- No animated diagrams
- No loading states or transitions
- No video embeds

**Why:** Static library, not interactive media

## The No-Diagram Test

Before adding a diagram, ask:

### 1. Would the essay work without it?
- [ ] **If yes:** Consider cutting the diagram
- [ ] **If no:** Diagram might be necessary

### 2. Does it reduce cognitive load?
- [ ] **If yes:** Good candidate for diagram
- [ ] **If no:** Explain in text instead

### 3. Can I explain this in 2-3 sentences?
- [ ] **If yes:** Use text, not diagram
- [ ] **If no:** Diagram might help

### 4. Is this inherently spatial or relational?
- [ ] **If yes:** Diagram appropriate
- [ ] **If no:** Text is clearer

### 5. Am I adding this to look professional?
- [ ] **If yes:** Delete it
- [ ] **If no:** Proceed with caution

**Rule:** When in doubt, leave it out. Text-only essays are fine.

## Creating Diagrams

### Tools

**Preferred:**
- **Excalidraw** - Hand-drawn style, perfect for this purpose
- **Figma** - If you need precise alignment, use minimal styling
- **Pen and paper â†’ scan** - If hand-drawn fits your style

**Avoid:**
- PowerPoint or Keynote (too formal)
- Canva (too decorative)
- Illustrator (too polished)

### Color Palette

**Use only:**
- **Black** (#000000 or #1C1917 stone-900) - Primary elements
- **Gray** (#78716C stone-500) - Secondary elements, labels
- **Indigo** (#4F46E5 indigo-600) - Accent, emphasis, highlights
- **White/Off-white** - Background

**Never use:**
- Multiple accent colors
- Bright or saturated colors for decoration
- Color gradients

### Size and Format

**Specifications:**
- Max width: 800px
- Format: PNG or SVG
- Aspect ratio: 16:9, 4:3, or square
- File size: Under 200KB

**Why these limits:**
- Loads fast
- Readable on mobile
- Doesn't dominate the page

### Placement

**In the essay:**
- After the text that introduces it
- Never before the context
- Surrounded by white space

**Not at:**
- Top of essay (start with text)
- Bottom as decoration
- Between unrelated paragraphs

## Maintenance

### Annual Review

Once per year, review all diagrams:

**Questions:**
1. Is this diagram still necessary?
2. Does it still accurately represent the system/process?
3. Is the style consistent with current guidelines?
4. Could it be simpler?

**Actions:**
- Update if system changed
- Simplify if possible
- Delete if no longer necessary
- Ensure consistent style across all diagrams

### Updating Diagrams

**When to update:**
- System or process changed
- You found a clearer way to show it
- Style doesn't match current guidelines

**When not to update:**
- Just to refresh the look
- To add more detail
- Because tools improved

**Remember:** Diagrams serve understanding, not aesthetics.

## Checklist: Before Adding a Diagram

- [ ] **Would the essay work without it?**
  - If yes: Consider cutting
  - If no: Proceed

- [ ] **Does it reduce cognitive load?**
  - If no: Use text instead
  - If yes: Proceed

- [ ] **Is this the most essential visual for this essay?**
  - Remember: Max 1 per essay
  - If there's something more important, use that instead

- [ ] **Is it hand-drawn style?**
  - Excalidraw or similar
  - Simple, clean, informal

- [ ] **Does it use minimal color?**
  - Black + gray + indigo only
  - No gradients, shadows, or decoration

- [ ] **Are labels clear and minimal?**
  - Short (1-5 words)
  - Specific, not generic
  - Readable size

- [ ] **Is it the right size?**
  - Max 800px wide
  - Under 200KB
  - PNG or SVG format

- [ ] **Is placement correct?**
  - After the text that introduces it
  - Not before context
  - Surrounded by white space

- [ ] **Is this explanatory, not performative?**
  - Shows how something works
  - Not there to impress
  - No stock imagery or decoration

---

## Summary

**Diagram guidelines:**
- Max 1 per essay
- Only when reduces cognitive load
- Hand-drawn style (Excalidraw)
- Minimal color (black, gray, indigo)
- Simple shapes and clean labels
- After introducing text, never before
- No stock photos, mockups, or decorative visuals
- Text must work without diagram

**Remember:**
Most essays don't need diagrams. When in doubt, leave it out.
